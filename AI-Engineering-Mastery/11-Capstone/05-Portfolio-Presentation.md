# Portfolio & Presentation Guide

## Portfolio Entry Template

For each cornerstone project and the capstone, create a portfolio entry:

```markdown
# [Project Name]

## Problem
[1-2 sentences]

## Architecture
![Architecture diagram]

## Key Technical Decisions
- **LLM Provider**: [Why this choice]
- **Vector Database**: [Why this choice]
- **Deployment**: [Where and how]

## Results
| Metric | Value |
|--------|-------|
| Faithfulness | 0.92 |
| Latency p95 | 2.1s |
| Cost per query | $0.03 |
| ... | ... |

## What I Learned
[3-5 bullet points]

## Links
- GitHub: [repo]
- Live demo: [URL]
- Design doc: [link]
```

## Demo Day Presentation

Format: 5 minutes, 3 slides

### Slide 1: Problem & Architecture (1 min)
- What does this system do?
- High-level architecture diagram
- One key design decision

### Slide 2: Demo (2 min)
- Live walkthrough of the system
- Show a complex query or edge case
- Show monitoring dashboard

### Slide 3: Results & Learnings (2 min)
- Key metrics (quality, latency, cost)
- What went wrong and how you fixed it
- What you'd do next

## Interview Talking Points

For each project, prepare answers to:
1. "What was the hardest technical challenge?"
2. "How did you evaluate whether it was working?"
3. "If you had 10x more data / users / budget, what would change?"
4. "Why did you choose [technology X] over [technology Y]?"
5. "What would you do differently if you rebuilt it?"
