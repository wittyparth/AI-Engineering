# Phase 11: Capstone Project

**Goal**: Combine everything you've learned into one production-grade system.

**Time**: 1-2 weeks  **Difficulty**: Hard  **Portfolio**: Yes (centerpiece)

## The Mission

Build and deploy a complete AI system that integrates ALL 10 previous phases:
- LLM Foundations → Multi-provider support
- Prompt Engineering → Systematic, versioned, tested
- Embeddings & Vector DB → Semantic search
- RAG → Production retrieval pipeline
- Advanced RAG → Agentic retrieval, query transformation
- AI Agents → Multi-agent orchestration
- Fine-Tuning → Domain expertise (optional but valued)
- Production Deployment → Docker, AWS, CI/CD
- Observability & Evaluation → Traces, metrics, evals, dashboards
- Advanced Topics → Security, guardrails, HITL

## Project Options

Choose ONE (or propose your own):

### Option 1: AI Customer Support System
- Multi-agent system (triage, research, response, review)
- RAG over product documentation
- Sentiment detection and escalation
- Human-in-the-loop for complex issues
- Observability dashboard + cost tracking
- Deployed with CI/CD

### Option 2: AI Code Review Assistant
- Agent that reviews PRs (code quality, security, style)
- RAG over your codebase and best practices
- Auto-fix suggestions for common issues
- GitHub App integration
- Evaluation suite (does it catch real bugs?)
- Deployed as GitHub Action + API

### Option 3: AI Research Platform
- Multi-agent research system (from Phase 6)
- Web search + document RAG + knowledge graph
- Report generation with citations
- User feedback loop
- Observability + cost management
- Production deployment

### Option 4: Your Own Idea
Propose your own project that covers:
- At least 6 of the 10 phases
- Real users (even if just yourself)
- Production deployment
- Evaluation and monitoring

## Capstone Process

```
Week 1:
  Mon: Project selection + design doc
  Tue-Wed: Core implementation
  Thu: Integration + evaluation

Week 2:
  Mon: Production deployment
  Tue: Monitoring + observability
  Wed: Polish + documentation
  Thu: Demo day prep
  Fri: Demo + retrospective
```

## Deliverables

1. **Design Doc** (1-2 pages): architecture, key decisions, tradeoffs
2. **Working System**: deployed, accessible, monitored
3. **Eval Report**: metrics showing system quality
4. **Cost Analysis**: per-query cost, monthly projection
5. **Post-Mortem**: what went wrong, what you'd change
6. **Demo**: 5-minute walkthrough of the system
7. **Portfolio Entry**: write-up for your portfolio
