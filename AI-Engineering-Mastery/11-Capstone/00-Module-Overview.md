# Capstone: Build Your Own Complex AI Application

**The Goal**: Design, build, deploy, and evaluate a production-grade AI application that is portfolio-worthy for senior AI engineering positions.

**Time**: ~20 hours (or ongoing — this is your portfolio piece)

## The Standard

This is NOT a tutorial project. This must meet the standard of AI systems deployed at companies like Klarna, Uber, Vodafone, and MindsDB. Your capstone should demonstrate:

1. **Production readiness** — deployed, monitored, cost-controlled
2. **Deep understanding** — you built key components from scratch before using frameworks
3. **Evaluation** — measurable quality metrics tracked over time
4. **Systems thinking** — handles failure, scales, is secure
5. **AI layer mastery** — you own the AI layer end-to-end

## Project Options

| Project | Complexity | Skills Demonstrated | Company Level |
|---------|-----------|-------------------|---------------|
| Enterprise RAG Platform | Hard | RAG, eval, deployment, cost optimization | Intercom, Notion |
| Multi-Agent Automation System | Hard | Agents, orchestration, HITL, MCP | Klarna, Uber |
| AI Observability & Eval Platform | Hard | Tracing, evaluation, dashboards | Langfuse, Logfire |
| AI Security & Guardrail System | Hard | Security, red-teaming, compliance | HiddenLayer, Protect AI |
| Multimodal Document Intelligence | Very Hard | Multimodal, RAG, agents, structured output | Glean, Hebbia |
| Your Own Idea | Varies | Whatever you design | Your choice |

## Requirements

Every capstone must include:
- [ ] Built from scratch (at least one key component)
- [ ] Framework used where appropriate
- [ ] Deployed to production (accessible endpoint)
- [ ] Cost tracking and budget limits
- [ ] Evaluation pipeline (CI gate)
- [ ] Monitoring dashboard
- [ ] Security guardrails
- [ ] Test suite (unit + integration + eval)
- [ ] Design document explaining tradeoffs
- [ ] Post-mortem: what broke, what you learned

## 🔴 Senior: Portfolio Quality

A senior AI engineer's portfolio doesn't show tutorials. It shows:
- Systems that ran in production
- Metrics that improved
- Problems that were solved
- Tradeoffs that were made

Your capstone should tell a story: "I built X, it solved Y, here's how I know it worked, here's what I'd do differently."
