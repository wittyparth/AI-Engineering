# AI Engineering Mastery

**The best AI engineering course in the world — built for engineers who build real systems.**

---

## Philosophy

This is NOT a tutorial course. Tutorials teach you to call APIs and memorize framework syntax. This course builds **AI engineering intelligence** — the ability to design, build, deploy, and debug production AI systems that solve real problems.

**The core insight:** Framework syntax changes every 6 months. But the underlying problems — retrieval quality, agent reliability, cost optimization, evaluation, observability — stay the same. This course teaches you the DEEP principles first, then shows how frameworks solve those problems. You'll understand WHY before you learn HOW.

---

## Structure: Application-Driven Learning

Every phase is organized around BUILDING a specific production-grade application. Frameworks appear when you understand the problem they solve — not as separate "add-on" files.

```
Phase 0  — AI Engineering Mindset (how senior engineers THINK)
Phase 1  — Build an LLM Gateway (Foundations + LiteLLM)
Phase 2  — Build a Prompt Engineering System (Prompting + DSPy)
Phase 3  — Build a Semantic Search Engine (Embeddings + Vector DBs)
Phase 4  — Build a Production RAG Engine (RAG + LangChain + LlamaIndex)
Phase 5  — Build an Advanced Knowledge System (Adv RAG + LlamaIndex Workflows)
Phase 6  — Build a Multi-Agent System (Agents + Pydantic AI + LangGraph + LiteLLM)
Phase 7  — Fine-Tuning & LLMOps (LoRA + Unsloth + Quantization)
Phase 8  — Deploy Everything to Production (vLLM + Llama Stack + AWS + CI/CD)
Phase 9  — Build an Observability & Evaluation Platform (Langfuse + RAGAS + Grafana)
Phase 10 — Advanced Topics (Outlines + Instructor + Multimodal + Security)
Phase 11 — Capstone (Your Own Production AI Application)
```

## Every Phase Follows The Build Loop

```
PROBLEM → BUILD RAW → ADD FRAMEWORKS → PRODUCTIONIZE → EVALUATE → DEBUG
```

## Prerequisites

- Python 3.11+ (advanced)
- FastAPI experience
- Docker & AWS familiarity
- Git & CLI proficiency
- You've shipped full-stack apps to production

## Learning Model

1. **Read** the concept (15-20 min)
2. **Build Raw** — implement without frameworks
3. **Add Framework** — see how it solves the pain
4. **Drill** — focused exercise (30 min)
5. **Productionize** — add monitoring, cost, security
6. **Evaluate** — measure quality metrics
7. **Debug** — find and fix failures
8. **Project** — portfolio-worthy system
9. **Gate** — pass the checkpoint before moving on
