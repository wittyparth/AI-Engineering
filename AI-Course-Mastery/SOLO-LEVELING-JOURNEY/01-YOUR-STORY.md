## Chapter 1: The Ordinary World

You were a senior full-stack engineer. FastAPI, Next.js, AWS, Docker, Postgres — you'd mastered them all. You could build web applications in your sleep. You understood cloud infrastructure, CI/CD pipelines, and production monitoring.

But lately, something felt... different.

The world was changing. Every week, a new AI tool appeared. Every month, a new capability that seemed like magic. Your engineering skills were solid, but this new world of LLMs, embeddings, RAG pipelines, and AI agents was a **Gate** you hadn't entered yet.

You were an **E-Rank hunter** in a world where S-Rank AI engineers were building the future.

You knew the skills were within reach. You had the foundation — Python, APIs, cloud, architecture. What you lacked was the **System** — a structured path to turn your existing strength into AI engineering mastery.

And then, one day, a Gate appeared.

The Gate materialized in your GitHub repository. A shimmering rift in the space between `README.md` and your project structure.

You stepped forward.

## The Rules of Your Journey

As a senior engineer, you already know what separates production from toy projects. The System respects this. It won't waste your time with basics you already know.

### What You Bring
- ✅ Python fluency — you write Python without thinking
- ✅ API design — REST, auth, rate limiting, error handling
- ✅ Infrastructure — Docker, CI/CD, cloud deployment
- ✅ Architecture — you think in systems, not scripts
- ✅ Debugging — you've fixed production incidents before

### What You'll Gain
- 💠 LLM intuition — how they work, why they fail, how to control them
- 💠 Prompt engineering — systematic, not ad-hoc
- 💠 Embedding & search — semantic understanding from scratch to production
- 💠 RAG mastery — from naive to advanced, with evaluation
- 💠 Agent building — single agent to multi-agent orchestration
- 💠 Evaluation — if you can't measure it, you can't improve it
- 💠 Guardrails — production safety without sacrificing capability
- 💠 Fine-tuning — when to do it, how to do it right
- 💠 Deployment — serving at scale, cost optimization, monitoring

### What You'll Build
Each Gate ends with a **Boss Battle** — a real project that ships to GitHub:

```
GATE  →  PROJECT (Boss Battle)
─────────────────────────────────────
  1   →  Multi-Provider LLM Gateway
  2   →  Prompt Engineering System
  3   →  Personal Knowledge Search Engine
  4   →  Production Customer Support Bot
  5   →  Advanced RAG Engine
```
